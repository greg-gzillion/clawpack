# dataclaw Local Library

## Purpose
This is the **local/personal library** for the **dataclaw** agent. 
It stores YOUR personal materials, not shared with other agents.

## Directory Structure

| Directory | Purpose | File Types |
|-----------|---------|------------|
| e_books/ | Personal e-books and PDFs | .pdf, .epub, .mobi |
| esearch/ | Research papers and articles | .pdf, .md, .txt |
| 
otes/ | Personal notes and observations | .md, .txt |
| citations/ | Saved personal citations | .md, .json |
| 	emplates/ | Custom document templates | .md, .txt, .docx |
| eferences/ | Personal reference materials | .md, .pdf |
| imports/ | Queue for new materials | Any |
| rchive/ | Archived old materials | Any |

## Relationship to Webclaw

- **Webclaw** = Global/shared references (all agents can read)
- **This library** = Local/personal (only you can access)

## Adding Materials

1. Place files in imports/ for automatic processing
2. Or add directly to appropriate category folder
3. Supported formats: .md, .txt, .pdf, .epub, .json

## Best Practices

- Use clear, descriptive filenames
- Add metadata as .json sidecar files when possible
- Tag files using #tags in content or filename
